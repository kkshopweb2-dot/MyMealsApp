Food Delivery App Documentation

User Guide

This guide provides instructions on how to use the Food Delivery application.

Getting Started

To start using the application, you first need to register for an account. Once registered, you can log in to access the dashboard and all the features.

  • Registration: Navigate to the signup page and fill in your details to create an account.
  • Login: Once you have an account, you can log in with your email and password.

Features

  • Dashboard: The main hub of the application where you can see an overview of your account and navigate to other features.
  • Order Meals: Browse through available meal plans and place an order.
  • File Complaints: If you have any issues with your order, you can raise a complaint.
  • Provide Feedback: Share your feedback on the food, delivery, and management.
  • Manage Delivery Locations: Add, update, and remove delivery addresses.
  • Set Meal Preferences: Specify your meal preferences, such as vegetarian, no nuts, etc.
  • Pause or Resume Meals: You can pause your meal plan for a certain period and resume it when you are back.
  • Renewal Payments: View your payment history and manage your subscription renewals.
  • Update Contact Information: Keep your contact details up to date.

Developer Guide

This guide is for developers who want to contribute to the project. It covers the project structure, setup, and API documentation.

Project Overview

The project is a monorepo containing three main parts:

  • Frontend (React): Located in the /src directory, this is the user-facing application.
  • Backend (Express.js): Located in the /backend directory, this is the main backend for the React application.
  • Backend (Laravel): Located in the /laravel directory, this is another backend that is currently used for database migrations and as a reference.

Backend (Express.js)

Setup

To run the Express.js backend, follow these steps:

cd backend
npm install
npm start

The server will start on http://localhost:5000.

Database

The backend uses a MySQL database named mymealsdb. The database schema is managed by the Laravel project's migrations. To set up the database, you need to configure the Laravel environment and run the migrations as described in the Laravel section.

Backend (Laravel)

Setup

The Laravel backend is primarily used for database migrations. To set it up, follow these steps:

cd laravel
composer install
cp .env.example .env
# Update the .env file with your database credentials for mymealsdb
php artisan key:generate
php artisan migrate
php artisan serve

The Laravel server will start on http://localhost:8000.

Frontend (React)

Setup

To run the React frontend, follow these steps:

npm install
npm start

The development server will start on http://localhost:3000.

Project Structure

The main source code for the React application is in the /src directory.

  • /src/api: Contains the configured Axios instance for making API calls.
  • /src/assets: Contains static assets like images.
  • /src/component: Contains reusable React components.
  • /src/css: Contains CSS files for styling.
  • /src/pages: Contains the main page components.
  • /src/redux: Contains the Redux store and slices for state management.
