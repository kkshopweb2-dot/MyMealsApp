Frontend (src)

src/api/axios.js

instance

Think of this as a pre-configured messenger. It knows the address of our backend server, so when other parts of the app want to send a message (like asking for data), they can use this instance without having to type the address every time. It also cleverly attaches a secret token to each message to prove that the request is coming from a logged-in user.

src/App.js

App

This is the main brain of our application's user interface. It acts like a traffic cop, directing users to the right pages based on the URL they visit. For example, if you go to /login, it shows you the login page. It also has a special power called "lazy loading," which means it only loads the code for a page when you actually need it, making the app start up faster.

Backend (backend)

backend/add_columns.sql

This is a script for the database, which is like a giant, organized spreadsheet. This particular script adds new columns to the "orders" sheet to store more information about each order, such as the customer's name, phone number, and payment details.

backend/controllers/authController.js

register

This function handles new user sign-ups. When a user creates an account, this code takes their password, scrambles it for security (a process called "hashing"), and then saves the new user's information to the database.

login

This function manages user logins. It checks if the email and password a user enters match what's stored in the database. If they match, it gives the user a special, temporary key (a "JWT token") that proves they are logged in.

backend/controllers/complaintController.js

These functions are the backend's tools for managing customer complaints. They allow the system to get a list of complaints, create a new one, view a specific one, update it, or delete it. They are the bridge between the user's complaint form and the database where complaints are stored.

backend/controllers/deliveryLocationController.js

This set of functions manages the delivery addresses for users. They handle tasks like saving a new address, fetching the list of saved addresses, and updating or deleting an existing one.

backend/controllers/feedbackController.js

These functions are responsible for handling customer feedback. They allow the system to save new feedback from a user, retrieve all feedback, or update and delete existing feedback entries in the database.

backend/controllers/mealController.js

getMeals

A simple function that fetches the list of all available meals from the database.

backend/controllers/mealPreferenceController.js

This group of functions manages a user's meal preferences. They handle saving a user's choices (like "vegetarian" or "no seafood"), retrieving their current preferences, and updating or deleting them.

backend/controllers/orderController.js

createOrder

This function is triggered when a user places a new order. It takes all the order details (like the plan, address, and payment info) and saves them to the database.

getOrders

This function retrieves a history of all orders for a specific user.

backend/controllers/orderMealController.js

getOrderMeals

This function is used to get information about which specific meals are included in which orders.

backend/controllers/pauseResumeMealController.js

These functions handle user requests to pause or resume their meal deliveries. They allow the system to record when a user wants to stop and restart their service, and for what reason.

backend/controllers/paymentController.js

This set of functions manages payment information. They are responsible for creating new payment records, retrieving payment history, and updating or deleting payment details.

backend/controllers/planController.js

getPlans

A straightforward function that fetches the list of all available subscription plans from the database.

backend/controllers/renewalPaymentController.js

These functions are specifically for handling subscription renewals. They manage the process of creating a new renewal payment, retrieving a history of renewals, and updating or deleting renewal records.

backend/controllers/userContactUpdateController.js

This group of functions manages requests from users to update their contact information. They handle creating a new update request, retrieving existing requests, and processing them.

backend/controllers/userController.js

getUsers

A function that retrieves a list of all registered users from the database. This is typically for administrative purposes.

backend/db.js

This file is the key that unlocks the database. It creates the connection between the backend code and the MySQL database, allowing the application to read and write data.

backend/middleware/verifyToken.js

verifyToken

This is a security guard for the backend. Before allowing access to certain private information, this middleware checks for the special key (the "JWT token") that was given to the user when they logged in. If the key is valid, the user can proceed. If not, they are denied access.

backend/routes/*.js

These files are like the road map for the backend. They define all the different URLs (or "routes") that the application understands. For example, they might define that a request to /api/users should be handled by the getUsers function.

backend/server.js

This is the main engine of the backend. It starts up the server, sets up all the necessary configurations (like allowing the frontend to communicate with it), and tells the server to use the road map defined in the routes files.
